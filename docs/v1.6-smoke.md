# V1.6 Smoke Checklist

## Signup Mode: open
1. Set `SIGNUP_MODE=open` in `.env` and restart dev server.
2. Go to `/signup`.
3. Step 1: enter name, email, password, locale → continue.
4. Step 2: enter org name + store name + phone → create account.
5. Confirm success state and verify hint appears.
6. Use the verification link (check server logs for the stub link) → `/verify/[token]`.
7. Login at `/login` with the new credentials.
8. Ensure redirect to `/onboarding` and onboarding page loads.

## Signup Mode: invite_only
1. Set `SIGNUP_MODE=invite_only` in `.env` and restart dev server.
2. Go to `/signup` and confirm “Request access” form appears.
3. Submit access request (no error).
4. Login as ADMIN and go to `/settings/users`.
5. Create invite for a new email + role.
6. Copy invite link and open `/invite/[token]` in a new session.
7. Accept invite: name + password + locale.
8. Login with invited account and verify access.

## Billing / Trial / Plan Limits
1. As ADMIN, open `/billing`.
2. Confirm plan, trial end date, usage, and limits render.
3. (Optional) Set trial to past in DB and verify mutations are blocked with `trialExpired` error.

## Tenant Isolation Spot Check
1. Create two orgs (use signup or DB seed).
2. As Org A admin, confirm Org B store/product/PO are not accessible.
3. Ensure cross-org inventory actions return forbidden/not found.

## RU/KG Locale Switching
1. On public auth pages (`/login`, `/signup`, `/invite/[token]`, `/reset`), switch locale via cookie or URL and verify labels change.
2. Inside app, switch locale and confirm UI language changes (nav, form labels).
